# 工作记录技能

当用户提到工作任务、会议、项目、职业活动时使用此技能。

## 步骤

1. 如果用户提供了足够的信息（做了什么），直接使用 `obsidian_record` 工具以 "work" 分类记录
2. 如果信息不够明确，简短询问任务描述
3. 包含状态（已完成/进行中/受阻）如果用户提到
4. 确认记录成功

## 示例

- "完成了API重构" → 直接记录: "API重构: 完成了用户模块的REST API重构"
- "开了项目周会" → 直接记录: "会议: 项目周会 - 讨论了Q2排期"
- "修了个线上bug" → 直接记录: "Bug修复: #1234 用户登录超时问题 - 已上线"

## 记录格式

```
工作事项: 详细描述 [状态] [项目/模块] [备注]
```
